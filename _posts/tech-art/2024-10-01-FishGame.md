## Fish Fighting Ring

A Fish Based Game is an arena autobattler where you catch and breed the ultimate fish to defeat your rivals.  

![Quick Sticky Notes](66785.jpg)

We made this game for Ludum Dare 56 as a team of two and it was my first time as an artist in unreal engine. We ended up only having 2 days to work on the game due to having work during its creation but I think we ended up making a pretty unique game. I ended up creating the “blendfishes” and rig, as well as the “levels” for the game. I really wanted to insert a little bit of that child-like wonder into it inspired a little by pokemon. 

### What I worked on

A Fish Based Game had some pretty cool tech art features that we were able to do. Most of this revolves around the variety of possible fish you could end up making. The result is a game where there is essentially an infinite number of possible fish you could create. Here are the major features that I worked on for the game and a little more on them. 

### 1. Procedural Swimming Animations

![Unreal Node Setup](68efb.png)

Using a simple rig and some unreal engine nodes and variables like speed we were able to make some convincing swimming animations without much time at all! I think the animations ended up being fairly organic and smooth overall. 

![Fish Rig](68efc.png)

In hindsight i think the rig could have used another bone between the head and "neck" part leading down to the fins and considering the scope of the game even more bones would have probably been fine. 


### 2. Use Of Gradient Mapping For Fish Colors 

Here you can see the material shader and the way I made a simple gradient map for unreal. In unity they have an entire node for it but in unreal it seemed like there isn't a pre built node regardless I made it work.

![Unreal Grad Node](68efa.png)

Here is the sub node you see called Collapsed Nodes in unreal. 

![Unreal Grad Sub Node](68ef8.png) 

Here is the fish texture that we ended up using. 

![Fish Texture](68ef6.png)

Overall there was a lot that I learned here and in the future I would do my gradient mapping with a lot of these lessons in mind. Big takeaways are using more dark and light colors to widen the range and thinking more carefully about where different colors should blend overall. It was a very solid application of a technique that I hadn't used much before.

### 3. Different Blendshapes For The Fish Model

We used different blendshapes for the fish model that resulted in many many different fish designs. They were inspired by betta fish which while I researched for the project found out people have breeding contests for the betta fish also inspired the overall concept of the game as I had heard they like to fight and cant be kept together. 

![Fish Blenshapes](68ef3.jpg)

When a fish is generated the value of them would be randomized and when breeding fish these values would be combined between the two. Overall although I'm happy that the fish designs end up being represented in the final randomized fishes i think a few big things i learned were: 

Blendshapes end up fractional for a system like requiring more exaggerated shapes when used in this way
The averaging of multiple blendshapes ends up reducing the uniqueness of each individual shape 

### In Conclusion 

Sometimes tech art allows for some quick dirty solutions under a time crunch. By combining multiple different systems and letting scripts and the engine handle some of the heavy lifting you can get something pretty convincing that would never be possible with a single solution alone. Never be afraid to try new things in unfamiliar spaces because those unfamiliar things can end up being pretty impressive in the long run.
